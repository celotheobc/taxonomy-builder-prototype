# Perspective Builder v5 — prototype notes

**Date:** 2026-06-30  
**Restore point before v5:** git commit `1257fd4` (v4 Process page + tube-map visualization)

## Summary

v5 is a targeted iteration on v4. The graph-first editing model is unchanged. The main shifts are panel semantics (right = inventory/configuration, bottom = diagnostics), richer cycle-resolution copy, and a second empty-state entry path to seed a perspective from an existing process.

## What changed

### Panel rebalance
- **Right inspector** now includes Included Objects and Included Event Sources as compact clickable lists (not tables).
- **Bottom panel** tabs are now: Relationships, Issues / Problems, AI Context.
- Perspective-level inspector also shows cache status, compact relationship/issue counts, and associated process when seeded from a process.

### Cycle resolution
- Scissors-on-edge graph interaction unchanged.
- Right inspector **Resolve cycle** cards now show mock **consequence previews** per removable edge (reachability, coverage %, routing notes).
- Issues / Problems tab still surfaces the cycle as a diagnostic; resolution controls stay in the right inspector and on the graph.

### Start from process (empty perspective)
- Empty state keeps search + suggested object chips.
- New section: **Or start from a process** with Order-to-Cash, Procure-to-Pay, and Accounts Payable chips.
- Seeds objects/events from process mock data and opens the normal graph editor (cycles use existing resolution flow).
- Process page **Create from entire process / Select subset** remains the place for informed subset selection (not duplicated on empty state).

### Versioning
- v4 preserved unchanged; selectable via cover card and L0 version picker (`v4`).
- v5 added as `v5` route; TheoBot remains on v4 only.

## Assumptions

- Consequence preview percentages and bullets are **mock only** (`cycleConsequencePreviews.js`).
- “Open asset” buttons are stubbed (prototype).
- Cache refresh updates timestamp locally only.

## Known issues / follow-ups

## Compare v4 vs v5

1. Open cover page → **version 4** vs **version 5**.
2. Or use L0 version picker: `v4` | `v5`.
3. **v4:** Included Objects/Events in bottom tabs; cycle cards without consequence bullets; no empty-state process start.
4. **v5:** Click canvas → objects/events in right inspector; bottom panel diagnostics only; cycle cards show consequences; empty state has process chips.
